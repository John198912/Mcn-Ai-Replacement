# 创作者服务平台权限申请与接入指南

> 平台独占数据是系统从"够用"到"强大"的分水岭。

---

## 一、为什么平台权限至关重要

| 独占数据 | 公开方式能获取？ | 对内容策略的价值 |
|---------|----------------|----------------|
| **完播率曲线**（秒级留存） | ❌ | 精准诊断内容哪个环节流失观众 |
| **流量来源构成** | ❌ | 判断是搜索驱动还是推荐驱动，决定SEO策略 |
| **粉丝画像**（年龄/性别/地域/兴趣） | ❌ | 精准匹配受众需求，调整内容深度和话题 |
| **同赛道对比数据** | ❌ | 了解自己在赛道中的位置 |
| **商单撮合** | ❌ | 获取品牌合作机会 |
| **创作者激励/收益** | ❌ | 收入分析和策略调整 |

**没有这些数据**：只能做粗颗粒度的"感觉式优化"  
**有了这些数据**：可以做精细到秒级的"数据驱动优化"

## 二、各平台申请路径

### 巨量星图（抖音/今日头条）

| 项目 | 说明 |
|------|------|
| 平台地址 | [star.toutiao.com](https://star.toutiao.com) |
| 创作者入驻条件 | 抖音粉丝≥1万 |
| 入驻方式 | 直接在星图创作者端注册，绑定抖音账号 |
| 可获取数据 | 星图账号信息、商单任务、投后数据 |
| API权限 | 在巨量引擎开放平台(open.oceanengine.com)注册开发者 → 企业认证 → 创建应用 → 申请接口 |
| API所需资质 | 企业营业执照 |
| 审批周期 | 企业认证3-5工作日，API审批5-7工作日 |
| 注意 | 2025年起审核趋严，需详细说明使用场景 |

**MVP阶段操作**：
1. 达到1万粉后立即入驻创作者端
2. 手动在星图后台查看/导出数据
3. 将数据手动录入飞书多维表格

**Phase 2 接入设计**：
- 申请API权限（需企业资质）
- Python脚本定时调用API → 自动同步到飞书

### 小红书蒲公英

| 项目 | 说明 |
|------|------|
| 平台地址 | [pgy.xiaohongshu.com](https://pgy.xiaohongshu.com) |
| 创作者入驻条件 | 粉丝≥1000 + 实名认证 + 年满18 + 无违规 |
| 入驻方式 | 在蒲公英平台申请开通"合作中心" |
| 可获取数据 | 笔记数据分析、粉丝画像、商单管理 |
| API权限 | 主要面向品牌方，创作者端无公开API |
| 变通方案 | 后台数据手动导出CSV/截图 + OCR解析 |

**MVP阶段操作**：
1. 达到1000粉后申请入驻蒲公英
2. 手动在后台查看笔记数据和粉丝画像
3. 关键数据截图/导出 → 录入飞书表格

## 三、优先级建议

| 优先级 | 平台 | 条件 | 理由 |
|--------|------|------|------|
| P0 | 小红书蒲公英 | 粉丝≥1000即可 | 门槛最低，最快可达 |
| P1 | 巨量星图 | 粉丝≥1万 | 数据最丰富，API最开放 |
| P1 | B站花火 | 粉丝≥1万 | API友好 |
| P2 | 视频号 | - | 生态封闭，价值有限 |

## 四、数据流转方案

```
各平台后台 ──(手动导出/截图)──→ CSV/图片
     │
     └──→ [数据解析] ──→ 飞书多维表格(content-review-schema)
              │
              └──→ Antigravity content-reviewer Skill分析
```

Phase 2 升级路径：
```
星图API ──(Python自动拉取)──→ 飞书多维表格
B站API ──(Python自动拉取)──→ 飞书多维表格
蒲公英 ──(RPA定时采集)──→ 飞书多维表格
```
