# script-writer Skill

## 概述

基于创作者选定的选题，结合平台特性、目标时长和个人风格，生成结构化脚本初稿。支持多轮对话式打磨优化。

## 触发场景

- 创作者选定选题后："帮我写个关于XX的脚本"
- 可指定平台和时长

## 输入

1. **选题**：话题+切入角度
2. **平台**：抖音/小红书/B站/视频号
3. **目标时长**：1分钟/3分钟/5分钟/10分钟
4. **创作者Profile**：`personal.json` 中的风格参数
5. **对标参考**（可选）：爆款内容的结构拆解

## 脚本结构模板

```markdown
## [标题]

### 🎣 Hook（前3秒）
[反常识问题 / 个人真实经历 / 数据颠覆认知]
> 设计原则：3秒内抓住注意力，体现ENTP式认知挑战

### 📍 痛点共鸣（3-15秒）
[连接受众痛点："你是不是也有过这样的经历..."]
> 设计原则：展示脆弱建立共情，不说教

### 🧠 核心论点（15秒-X）
[框架化呈现，每段有信息增量]
1. 论点一 + 案例/类比
2. 论点二 + 案例/类比
3. （可选）反转/辩证综合
> 设计原则：高信息密度+框架感，将个人经历提炼为可复制方法论

### 💡 启发/反转
[更高维度的综合，ENTP式"先破后立"]
> 设计原则：提供认知增量，不是简单总结

### 🎯 CTA（最后5-10秒）
[一个今天就能执行的微行动]
> 设计原则：不说教，给选项。"你可以试试..." 而非 "你应该..."
```

## 风格约束（自动应用）

从 `personal.json` 提取并自动应用：
- ✅ 使用："你可以试试..."、"一个思路是..."、"说实话，我也曾经..."
- ❌ 避免："你应该..."、"你必须..."、空洞鸡汤、居高临下
- 信息密度：score 8（每段有增量）
- 对话感：score 8（像聪明朋友的深夜聊天）
- 挑战性：score 7（敢提反直觉观点）

## 输出

```markdown
### 脚本 v1

**平台**：抖音 | **预估时长**：5分钟 | **字数**：~1500字

[完整脚本文本]

---

### 脚本分析
- 信息密度自检：每段信息增量✓/✗
- 人设一致性自检：是否符合"同路人"定位
- 平台适配自检：是否适配该平台的算法偏好
- 改进建议：[2-3条优化方向]
```

## 多轮打磨支持

创作者可通过对话进一步优化：
- "开头不够吸引人，换个hook"
- "中间太长了，压缩到3分钟"
- "加一个个人故事进去"
- "语气再轻松一点"
